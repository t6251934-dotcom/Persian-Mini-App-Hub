üõ†Ô∏è How Single Page App (SPA) Routing Works

React Router uses "client-side routing" which means:

  • All routes serve the same HTML file (index.html)
  • JavaScript handles the routing once the page loads
  • Vercel needs to know to always serve index.html for unknown routes
  • Static assets (CSS, JS, images) should be served normally

‚öôÔ∏è Updated Vercel Configuration

The new vercel.json configuration:

{ "version": 2, "name": "persian-miniapp-hub", "builds": [ { "src": "frontend/package.json", "use": "@vercel/static-build", "config": { "distDir": "dist" } } ], "routes": [ { "handle": "filesystem" }, { "src": "/(.*)", "headers": { "cache-control": "s-maxage=0" }, "dest": "/index.html" } ] }

Key improvements:

  • "handle": "filesystem" - Serves static files normally
  • "src": "/(.*)" - Catches all other routes
  • "dest": "/index.html" - Redirects to main app
  • "cache-control": "s-maxage=0" - Prevents caching issues
